# ⚙️ Los campos clave que debes conocer

Cuando entras a [start.spring.io](https://start.spring.io),
verás varias opciones importantes para configurar tu proyecto Spring Boot.

---

# 📚 Secciones principales

| Configuración | Descripción |
|---|---|
| 📦 [***Project***](./project.md) | Define el gestor de dependencias (`Maven` o `Gradle`). |
| ☕ [***Language***](./language.md) | Selecciona el lenguaje del proyecto (`Java`). |
| 🌱 [***Spring Boot Version***](./version.md) | Permite elegir la versión de Spring Boot. |
| 🏷️ [***Project Metadata***](./metadata.md) | Configura el nombre y datos básicos del proyecto. |
| 📚 [***La sección de Dependencies***](./Dependencies.md) | Agrega funcionalidades mediante Starters. |

---

# 🚀 ¿Por qué es el mejor amigo del desarrollador?

---

## 🧩 Reduce la complejidad

No necesitas:

- buscar librerías manualmente,
- configurar dependencias una por una,
- ni resolver conflictos entre versiones.

Spring Initializr prepara todo automáticamente.

---

## ✅ Cero errores de inicio

El proyecto descargado normalmente:

- compila,
- inicia,
- funciona desde el primer momento.

---

## ⚙️ Configuración automática

Spring deja listo el entorno para que anotaciones como:

```java id="init-anno-1"
@Service
@Repository
@RestController
```

funcionen apenas empiezas a programar.

---

## 🎮 Analogía para recordar

Piensa en Spring Initializr como el:

### 🕹️ “Creador de personajes” de un videojuego

Antes de comenzar eliges:

- tu clase (Spring Web),
- tus habilidades (JPA, Security),
- y tu nombre (Artifact).

# 🏷️ Project Metadata

Project Metadata son los datos básicos que identifican tu proyecto dentro de Spring Boot.

Estos valores ayudan a:

- organizar la aplicación.
- identificar el proyecto.
- generar paquetes.
- y construir correctamente el sistema.

---

# 📚 Elementos principales

| Configuración | Función |
|---|---|
| 🏢 [***Group***](./Group.md) | Define la organización o paquete principal del proyecto. |
| 📦 [***Artifact***](./Artifact.md) | Define el nombre de la aplicación. |
| 📁 [***Packaging***](./Packaging.md) | Define el formato final del proyecto (`Jar` o `War`). |

---

# 🧠 ¿Por qué es importante?

Estos datos permiten que Spring Boot:

- genere automáticamente la estructura del proyecto,
- organice los paquetes correctamente,
- prepare la aplicación para compilarse y ejecutarse.

---

# 💡 Idea rápida para recordar

Piensa en Project Metadata como la:

    ## 🪪 “Identidad” de tu aplicación

    Aquí defines:

        - quién eres (`Group`),
        - cómo te llamas (`Artifact`),
        - y cómo será empaquetado el proyecto (`Packaging`).

---

# 📌 Resumen rápido

| Concepto | Significado |
|---|---|
| Project Metadata | Información básica del proyecto |
| Incluye | `Group`, `Artifact` y `Packaging` |
| Función principal | Identificar y organizar la aplicación |
| Importancia | Configuración inicial del proyecto |

# 🌱 Spring Boot Version

Aquí eliges la versión de Spring Boot que utilizará tu proyecto.

La versión seleccionada define:

- las funcionalidades disponibles,
- la compatibilidad de librerías,
- las mejoras de rendimiento,
- las características del framework.

---

# 📌 Recomendación

Usa siempre una versión que sea:

- ✅ estable,
- 🚀 reciente,
- y ❌ que NO diga:
  - `Snapshot`
  - `M`
  - `RC`

---

# ⚠️ ¿Qué significan esas etiquetas?

| Etiqueta | Significado |
|---|---|
| `Snapshot` | Versión en desarrollo |
| `M` | Milestone (versión de prueba) |
| `RC` | Release Candidate (casi final) |

Estas versiones pueden contener:

- errores.
- cambios inesperados.
- funciones inestables.

---

# ✅ ¿Por qué usar versiones estables?

Porque garantizan:

- 🔒 mayor estabilidad,
- 🧩 mejor compatibilidad,
- 📚 más documentación,
- 🛠️ menos errores inesperados.

---

# 💡 Consejo para principiantes

Si estás aprendiendo Spring Boot:

👉 Usa siempre la versión estable más reciente disponible en Spring Initializr.

Así tendrás:

- mejores herramientas.
- soporte actualizado.
- ejemplos más compatibles con internet.

---

# 📌 Resumen rápido

| Concepto | Significado |
|---|---|
| Spring Boot Version | Versión del framework Spring Boot |
| Recomendación | Usar versiones estables |
| Evitar | `Snapshot`, `M` y `RC` |
| Beneficio principal | Mayor estabilidad y compatibilidad |

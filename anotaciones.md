# Las Anotaciones

Si los Beans son los componentes de nuestra aplicación, las Anotaciones son las etiquetas que pegamos sobre ellos para decirle a Spring qué función cumplen.

> “Las anotaciones son una forma de agregar metadatos al código. No afectan directamente el comportamiento lógico, pero herramientas como Spring las usan para realizar operaciones específicas.”

En términos simples:

> 🏷️ Una anotación es una orden para Spring.

---

# 1. ¿Cómo funcionan en la práctica?

Imagina que escribes una clase llamada `Calculadora`. Para Java, es solo una clase más. Pero si le pones la anotación `@Component` encima, le estás diciendo a Spring:

> “Oye, registra esta clase, crea un objeto de ella y guárdalo en tu contenedor porque lo voy a necesitar después”.

Spring hace un uso masivo de estas etiquetas para simplificar la configuración.

Antes, para configurar una aplicación, tenías que escribir cientos de líneas en archivos de texto aburridos (`XML`); ahora, solo pones una “etiqueta” sobre tu código.
